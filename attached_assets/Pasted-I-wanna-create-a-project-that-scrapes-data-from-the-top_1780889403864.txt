That's actually one of the easiest parts.
Most companies don't build their own careers site. Their careers page eventually redirects to a third-party ATS (Applicant Tracking System).

Examples:

ATS	URL Pattern
Workday	myworkdayjobs.com
Greenhouse	boards.greenhouse.io
Lever	jobs.lever.co
iCIMS	careers.icims.com
SmartRecruiters	jobs.smartrecruiters.com
Ashby	jobs.ashbyhq.com
Jobvite	jobs.jobvite.com
For example:
https://www.rsmus.com/careers
might redirect to:
https://rsm.wd5.myworkdayjobs.com/Experienced
Once you discover that URL once, you can store:
firm_name
careers_page
ats_type
ats_url
Example:
RSM
https://www.rsmus.com/careers
WORKDAY
https://rsm.wd5.myworkdayjobs.com
Then every day you query the ATS directly instead of scraping the marketing website.
A neat trick:
Many ATS providers expose APIs.

Workday
Workday job listings are usually fetched through JSON endpoints.
Example:

https://company.wd5.myworkdayjobs.com/wday/cxs/company/jobs/jobs
returns JSON.
Greenhouse
Greenhouse literally has a public API:
https://boards-api.greenhouse.io/v1/boards/company/jobs
Lever
https://api.lever.co/v0/postings/company
returns JSON.
Ashby
Many Ashby boards expose structured JSON.
So the architecture becomes:
Top 500 CPA Firms
        ↓
Discover ATS once
        ↓
Store ATS type
        ↓
Daily API pulls
        ↓
Normalize jobs
        ↓
Filter internships
        ↓
Newsletter
This is much more reliable than scraping HTML.

Phase 3: ATS Discovery Script
Write a script that visits each careers_url and detects where it redirects.
Logic:

if "myworkdayjobs.com" in final_url:
    ats_type = "workday"
elif "greenhouse.io" in final_url:
    ats_type = "greenhouse"
elif "lever.co" in final_url:
    ats_type = "lever"
elif "icims.com" in final_url:
    ats_type = "icims"
else:
    ats_type = "custom"
Use Playwright because some careers pages load links with JavaScript.

A couple of notes:
Some firms have merged recently (e.g., Forvis + Mazars, DHG history), so you'll want your scraper to handle redirects and acquisitions gracefully.
The careers_url field doesn't need to be perfect. Your ATS discovery script will visit the page and determine the actual job board URL.
Add these fields now even if they're empty:
ats_type
ats_url
last_checked
internships_found
Your first script should simply:
For each firm:
  Visit careers_url
  Follow redirects
  Record final URL
  Detect ATS provider
  Save ats_type + ats_url
After that, the whole project becomes dramatically easier because you're scraping maybe 5–10 ATS providers instead of 500 unique websites. That's the key architectural insight that makes this project scalable.
